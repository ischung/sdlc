#### Logic — 데이터 처리 흐름

1. 사용자가 [진입점]에서 [행동]을 수행한다
2. 시스템은 [조건 A]를 확인하고

#### Validation — 입력값 검증 규칙

| 필드 | 필수 여부 | 형식 / 제약 | 실패 시 메시지 |
|------|-----------|-------------|----------------|
| [필드명] | 필수 | [예: 이메일 형식, 최대 255자] | "유효한 이메일을 입력해주세요." |

| [필드명] | 필수 | [예: 8자 이상, 영문+숫자] | "비밀번호는 영문과 숫자를 포함해 8자 이상이어야 합니다." |

#### Edge Case — 예외 상황 처리

| 케이스 | 발생 조건 | 기대 동작 |
|--------|-----------|-----------|
| 데이터 없음 | 검색 결과 0건 | Empty State UI + 대안 행동 제시 |
| 네트워크 지연 | 응답 3초 초과 | 스켈레톤 UI → 5초 초과 시 재시도 안내 |

| 중복 입력 | 동일 데이터 재제출 | "이미 등록된 항목입니다" 안내 + 기존 항목으로 이동 |

## 6. UI/UX 요건 (Design Spec)

### 6-1. 레이아웃

| 항목 | 요구 사항 |
|------|-----------|
| **화면 구성** | [예: 상단 헤더 / 좌측 메뉴 / 중앙 콘텐츠 / 하단 CTA] |
| **정보 위계** | [예: 핵심 정보는 Above the fold, 보조 정보는 펼침 영역] |
| **반응형** | 모바일 (≥ 375px) / 태블릿 (≥ 768px) / 데스크톱 (≥ 1024px) 대응 필수 |
| **접근성** | WCAG AA 준수, 키보드 내비게이션 지원 |

### 6-2. 인터랙션 디테일

- **애니메이션**: 진입/퇴장 전환은 200~300ms, `ease-out`
- **모달 등장**: 배경 딤 처리 + 중앙 확대 등장, `esc`로 닫기 지원
- **로딩**: 콘텐츠 1초 이내 로딩 시 스피너 생략, 1초 초과 시 스켈레톤 표시
- **피드백**: 상태 변화(성공/실패)는 토스트 메시지로 알림, 3초 후 자동 소멸
- **마이크로 인터랙션**: 버튼 hover/active 상태 구분, 클릭 시 ripple 효과 (선택)

### 6-3. 카피라이팅 (Copywriting)

> 버튼·안내 문구·에러 메시지는 **사용자 관점의 언어**로 통일합니다.
> 모호한 기술 용어(예: "토큰이 만료되었습니다") 대신 **다음 행동을 알려주는** 문구를 사용합니다.

#### 주요 버튼 레이블

| 맥락 | 레이블 | 비고 |
|------|--------|------|
| 저장 액션 | "저장하기" | "확인"은 모호하므로 지양 |
| 취소 액션 | "취소" | |
| 파괴적 액션 | "삭제하기" | 확인 모달 필수 |
| 진행 액션 | "다음" / "시작하기" | 플로우 길이에 따라 선택 |

#### 안내 문구 (Empty / Success / Info)

| 상황 | 문구 |
|------|------|
| 빈 목록 | "아직 등록된 항목이 없어요. 첫 항목을 추가해볼까요?" |
| 저장 성공 | "저장했어요." |
| 자동 저장 | "자동으로 저장 중..." |

#### 에러 메시지

| 에러 유형 | 문구 | 원칙 |
|-----------|------|------|
| 네트워크 오류 | "연결이 불안정해요. 잠시 후 다시 시도해주세요." | 원인 + 다음 행동 |
| 유효성 실패 | "[필드명] 형식을 확인해주세요." | 어느 필드인지 명시 |
| 권한 부족 | "이 기능은 관리자만 이용할 수 있어요." | 책망 금지, 차분한 톤 |
| 서버 오류 | "일시적인 문제가 발생했어요. 곧 복구할게요." | 사용자 탓으로 들리지 않게 |

> 기능 요건이 **무엇을 하는가**라면, 비기능 요건은 **얼마나 잘 해야 하는가**입니다.
> 측정 가능한 기준(수치)으로 기술하세요.

### 7-1. 환경 (Environment)

| 항목 | 요구 사항 |
|------|-----------|
| **지원 브라우저** | Chrome / Safari / Edge / Firefox 최신 2개 메이저 버전 |
| **지원 OS** | Windows 10+, macOS 12+, iOS 15+, Android 10+ |
| **최소 해상도** | 모바일 375px, 데스크톱 1280px |
| **네트워크** | 3G 환경에서도 핵심 동작 가능 (기본 기능 우선 렌더) |

### 7-2. 성능 (Performance)

| 지표 | 목표 값 | 측정 조건 |
|------|---------|-----------|
| 초기 로딩 (LCP) | **≤ 2.0초** | 데스크톱 Fast 3G |
| 상호작용 지연 (INP) | **≤ 200ms** | p75 기준 |
| API p95 응답 | **≤ 300ms** | 프로덕션 RUM |
| 동시 접속 처리량 | **≥ [N] RPS** | 피크 시간 기준 |
| 가용성 (SLA) | **≥ 99.9%** | 월간 기준 |

### 7-3. 보안 (Security)

- [ ] **전송 구간 암호화**: 모든 통신 TLS 1.2 이상
- [ ] **저장 암호화**: 개인정보 / 결제정보 AES-256 암호화
- [ ] **권한 관리**: 역할 기반 접근 제어(RBAC) 적용
- [ ] **인증**: 세션 만료·토큰 갱신 정책 수립
- [ ] **감사 로그**: 민감 작업(삭제, 권한 변경)은 사용자·시각·IP 기록
- [ ] **컴플라이언스**: [예: 개인정보보호법, GDPR, PCI-DSS 해당 여부]
